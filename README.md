school-gradebook-qa-tests

Manual QA project for testing an electronic school gradebook. Includes test cases for login functionality, assessment management, boundary value analysis of the comment field, and pairwise testing of the support form. Demonstrates practical testing techniques and structured QA documentation

Test_Suites:
Comment field - Boundary Value Analysis. 
Check the comment field using the boundary technique

Electronic journal - Working with Assessments. 
Checking the teacher's ability to work with grades in the nz.ua electronic journal: give new grades, view existing ones, edit and delete grades, add comments to grades and notes.

Login Page Testing. 
This test suite verifies the login functionality on the nz.ua portal. The test cases are designed to cover valid and invalid credential scenarios, input validation, UI behavior, security input handling, and session persistence. Techniques used include boundary value analysis, equivalence partitioning, and negative testing.

Pairwise Testing of the Technical Support Contact Form Submission. 
This test suite verifies the filling and submission of the technical support contact form.
The test cases are designed using the Pairwise Testing test design technique.
They cover validation of the following input fields: Full Name, Phone Number, Email Address, Position, School, and Message Text.
